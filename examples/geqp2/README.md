# Example: geqp2

In this example, we compute the QR factorization of a matrix filled with random numbers using Drmac's algorithm.
There are 5 primary functions, `permute_matrix`, `check`, `printMatrix`, `run`, and `main`.
In the run function:

    A is a m-by-n matrix filled with random numbers.

    Matrix A_orig is the original copy of Matrix A which will be used in the check function later

    Matrices S (m, r) and C (r, n) are created and filled with zeros which later multiply together to create A

    The slices used in the example are created from Matrix A and vector Tau.

    Slice A_1 (m, k) is the leading block while slice A_2 (m, n-k) is the trailing block

    A standard Householder QR is performed on the first k columns using tlapack::geqr2 on A_1

    After obtaining the orthogonal factors from A_1, they are then applied from the left to the remaining columns on the trailing block A_2 using tlapack::unmqr. This will update the upper block R_12 (k, n-k) and leaves a remaining unreduced trailing block A_22 (m-k, n-k).

    Drmac's algorithm is then called on the trailing matrix sub-block A_22 to capture the remaining rank space.

    The columns of the upper right block R_12 are permuted to match the active swaps made inside the bottom submatrix, preserving the overall matrix consistency.

    The numerical rank is determined by sweeping the main diagonal elements of the combined upper-triangular matrix R and counting elements that stay above an absolute threshold bounded by machine precision.

    The QR Factorization of A produced through Drmac's algorithm is then checked against a QR factorization produced by geqr2

In the check function:

    The Frobenius norm of the original matrix A is computed using tlapack::lange to serve as a baseline denominator for relative error verification.

    An n×n matrix R is allocated, initialized to zero with tlapack::laset, and populated by copying the upper-triangular output factor from the factored matrix using tlapack::lacpy.

    The explicit m×n orthogonal matrix Q is generated by extracting the Householder vectors via tlapack::lacpy and reconstructing the explicit basis with tlapack::ung2r.

    The reconstruction accuracy is evaluated by performing the matrix multiplication A=A−Q×R via tlapack::gemm and taking its Frobenius norm to compute the relative residual error||A - Q*R||_F / normA​.

    The orthogonality error of the unitary factor is evaluated by checking ||I - Q^H Q||_F through an identity generation step and a conjugate-transpose tlapack::gemm sequence.

In the permute_matrix function:

    An in-place cycle-following algorithm is executed to rearrange either the rows or columns of a matrix according to a specified index configuration.

    It iterates across the primary dimension using a tracking array to safely swap full vector segments via tlapack::swap.

In the printMatrix function:

    The dimensions are parsed to print out the components row-by-row onto the standard system output.
    
The code uses the routine [tlapack::geqp2](../../include/lapack/geqp2.hpp) to perform the complete factorization in place, and [tlapack::ung2r](../../include/lapack/ung2r.hpp) to generate the m-by-n matrix Q. We store R in a n-by-n upper triangular matrix.

## Build

We provide two options for building this example:

1. Following the standard CMake recipe

```sh
mkdir build
cmake -B build      # configuration step
cmake --build build # build step
```

You will find the executable inside the `build` directory.

2. Using `make` on the same directory of [example_geqp2.cpp](example_geqp2.cpp). In this case, you may need to edit `make.inc` to set the environment variables needed by [Makefile](Makefile). After a successful build, the executable will be in the current directory.

## Run

You can run the executable from the command line. You can pass up to 4 integer arguments to the executable. The first one sets `m`, the second one sets `n`, the third one sets `r`, and the fourth one sets `k`. Their default values are `m = 7; n = 5; r = 4; k = 2;` for no particular reason.
